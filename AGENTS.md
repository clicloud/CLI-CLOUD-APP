`/AGENTS.md` is the source of truth for repository instructions.

Any assistant, agent, or contributor working in this repository must read this file before making changes.

## Pull Request Rules

- Keep every pull request to one concept only.
- Prefer multiple small pull requests over one broad pull request.
- Make the smallest working change that solves the task.
- Do not mix refactors, cleanup, or unrelated polish into the same pull request.
- If a request naturally spans multiple concepts, stop and split it into separate branches/pull requests before coding.
- Keep pull request titles and descriptions short, factual, and scoped to the exact change.
- Before committing, verify that all staged files belong to the same concept.
- If staged changes cover more than one concept, split them.
