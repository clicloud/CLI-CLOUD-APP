## Telegram — Channel Structure Proposal

Based on repeated feedback about losing track of tasks and updates in the main chat, proposing:

#cli-general — Main discussion, casual chat
#cli-tasks — Task assignments and status updates (Clide-managed)
#cli-updates — Weekly progress, announcements, milestones
#cli-feedback — Community feedback, bug reports, feature requests
#cli-deploy — Container deployment discussion and troubleshooting

This keeps signal high in each channel and makes it easy to catch up without scrolling through 200 messages of general chat.
